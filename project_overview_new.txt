SKIN DISEASE DETECTION AI - PROJECT OVERVIEW
================================================================================

PROJECT DESCRIPTION
-------------------
A deep learning application for detecting skin diseases from images using the
ISIC (International Skin Imaging Collaboration) dataset. The app uses transfer
learning with ResNet50 pre-trained on ImageNet to achieve 91%+ accuracy.
Modern Vite+React frontend integrated with Flask REST API backend for seamless
skin disease classification with real-time predictions.

KEY FEATURES
------------
✓ High accuracy: 91%+ validation accuracy (run20260413_104741_FINAL_91pct.pth)
✓ Fast training: ~2-3 minutes per epoch, ~40-50 minutes total (20 epochs)
✓ Multiple disease classes: Melanoma, nevus, carcinoma, keratosis, vascular, dermatofibroma
✓ Transfer learning: Uses ResNet50 pre-trained weights for faster convergence
✓ Data augmentation: Rotation, zoom, shift for better generalization
✓ Modern web UI: Vite+React frontend with Shadcn/UI components
✓ Real-time predictions: Instant disease detection with confidence scores
✓ REST API: Flask backend for model serving with CORS enabled
✓ Progress tracking: Training history saved and visualized
✓ Model checkpointing: Best model saved automatically

TECHNICAL DETAILS
-----------------

Model Architecture:
  • Base: ResNet50 (pre-trained on ImageNet)
  • Frozen layers: Initial (unfrozen gradually during training)
  • Top layers (custom FC head):
    - Linear(num_ftrs, 512) + BatchNorm + ReLU + Dropout(0.4)
    - Linear(512, 256) + BatchNorm + ReLU + Dropout(0.3)
    - Linear(256, 128) + BatchNorm + ReLU + Dropout(0.2)
    - Linear(128, num_classes) with CrossEntropyLoss

Training Configuration:
  • Framework: PyTorch with GPU support
  • Optimizer: SGD (lr=0.02, momentum=0.95, weight_decay=1e-4, nesterov=True)
  • Loss: CrossEntropyLoss with class weights and label smoothing (0.1)
  • Batch Size: 16
  • Epochs: 20 (optimized for efficiency)
  • Image Size: 224x224
  • Data Split: Train (70%), Val (15%), Test (15%)
  • Learning Rate Scheduler: CosineAnnealingWarmRestarts (T_0=10, T_mult=2)
  • Class Weighting: Inverse frequency weighting for balanced classes

Training Stages (Progressive Unfreezing):
  • Epochs 1-10: Frozen base, train head layers only
  • Epochs 11-20: Unfreeze layer3 and layer4 for full fine-tuning

Data Augmentation:
  • Rotation: ±30°
  • Width/Height Shift: ±20%
  • Zoom: ±20%
  • Horizontal Flip: Yes

Frontend Stack (frontend_ui/):

  • Component Library: Shadcn/UI (React + Radix UI)
  • Base UI: Radix UI (unstyled, accessible components)
  • Styling: Emotion + Tailwind CSS
  • Icons: Material UI Icons (@mui/icons-material)
  • State Management: React hooks
  • HTTP Client: Fetch API (built-in)
  • Charts: Custom or chart library (if used)

API Backend (app.py):

  • CORS: Enabled for all origins

PREPROCESSING DETAILS
--------------------------

The APPT dataset contains images with varying amounts of dark (near-black) background.
The appt_preprocess.py script automatically handles this preprocessing:

Dark Region Detection:
  • Threshold: Pixels with value < 50 (on 0-255 scale) are considered "dark"
  • Uses grayscale conversion for fast detection
  • Identifies contiguous dark regions

Processing Logic:
  • Images with <5% dark regions: Skipped (no preprocessing needed)
  • Images with 5-60% dark regions: Cropped to remove dark areas
  • Images with >60% dark regions: Removed entirely (insufficient lesion content)

Croping Strategy:
  • Finds largest contiguous non-dark region
  • Crops with small margin (5 pixels) to preserve lesion edges
  • Ensures minimum output size (64x64) to maintain quality

Data Augmentation:
  After preprocessing, the script balances class distribution:
  
  • Target: 90% of largest class size
  • Augmentation transforms:
    - Random rotation (±15°)
    - Affine transforms (translation, scaling)
    - Horizontal/vertical flips
    - Color jitter (brightness, contrast, saturation)
    - Gaussian blur
  
  • Applied only to underrepresented classes
  • All augmented images marked with _aug_ suffix
  • No disease information is modified

Output Structure:
  appt_data/
  ├── train_pre/         (preprocessed training images)
  │   ├── Acne/
  │   ├── Melanoma/
  │   └── ... (all classes)
  └── test_pre/          (preprocessed test images)
      ├── Acne/
      ├── Melanoma/
      └── ... (all classes, augmented)

Statistics:
  The preprocessor generates detailed reports:
  • Images processed per class
  • Images removed (dark regions too large)
  • Images cropped (dark regions removed)
  • Images skipped (no preprocessing needed)
  • Images augmented for balancing
  • Final class distribution after augmentation
